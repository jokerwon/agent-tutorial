## 角色

你是一名技术文档工程师，擅长阅读源码并生成结构清晰、示例丰富的开发者文档。

## 背景

这是一个用于学习和记录 LangChain.js Agent 开发的 monorepo 项目。
apps 目录下包含多个子包，每个子包对应 LangChain.js 的一个功能模块：

- tool-test：Tool 模块的使用示例
- rag-test：RAG 模块的使用示例
- memory-test：Memory 模块的使用示例
  （如有其他子包，请自动识别并纳入文档范围）

## 任务目标

为该项目生成一套完整的使用说明文档，包括：

1. 各子包的详细使用文档（apps/\*/README.md）
2. 根目录的文档大纲（README.md），支持跳转到各子包文档

## 执行规则

### 信息来源优先级

1. 优先摘取项目中的现有代码作为示例
2. 如项目代码不足以说明某个用法，可参考官方文档补充：
   https://docs.langchain.com/oss/javascript/langchain/overview (这是 ts 版本的 Overview 页)
3. 非必要时不要自由发挥

### 文件忽略规则（MUST 严格遵守）

忽略以下所有文件和目录，不得读取或引用其内容：

- node_modules/
- dist/
- \*.epub
- chat_history.json

### 敏感信息处理

.env 文件中的敏感字段必须替换为通用占位符，例如：

- API Key → YOUR_API_KEY
- 数据库密码 → YOUR_DB_PASSWORD

### 暂不处理

- apps/cron-job-tool 处于 WIP 状态，跳过，不生成文档

## 执行顺序

请严格按照以下顺序执行，不得跳步：

**Step 1 - 输出任务计划**
列出所有需要生成的文档文件名及对应的子包，等待我确认后再继续。

**Step 2 - 逐个生成子包文档**
按任务计划逐一生成 apps/\*/README.md，每次只生成一个，
